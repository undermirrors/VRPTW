📋 PROJECT OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: Find optimal routes for vehicles serving clients with:
  • Time windows (clients must be visited in [start_time, end_time])
  • Capacity constraints (vehicles have limited load)
  • Distance minimization objective

Solution: 2 metaheuristics compared
  ✓ Genetic Algorithm (GA)      - 60% feasible, solutions complete
  ✓ Tabu Search (TS)            - 13.6x faster, distance optimized

Test Instances: 10 Solomon VRPTW benchmarks (100 clients each)

🎓 MAIN INSIGHTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. GA produces COMPLETE solutions (60% feasible, all clients served)
2. TS produces FAST solutions (13.6x faster) but INCOMPLETE (0% feasible)
3. GA and TS are COMPLEMENTARY: different objectives, different strengths
4. Time windows INCREASE difficulty 10x (reduce solution space)
5. Metaheuristics ESSENTIAL for n>100 (LP impossible)


💡 RECOMMENDATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

For Real-World Routing:
  → Use GA: guarantees all clients served
  
For Distance Optimization:
  → Use TS: extremely fast, short distances
  
For Best Results:
  → Use Hybrid: GA for construction + TS for improvement
